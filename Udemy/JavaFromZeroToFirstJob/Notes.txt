58. Enums
-----------
• What is an Enum type?
  + Is a special dataType taht enables for a variable to be a set of predefined constants.
  + An Enum type must always be equal to one of the values predefined for it
  + You need to know all possible values at compile-time
  + All values are constants, therefor named with capital letters
  + Each value is associated with an ordinal number, listed in strict sequence
    - can be checked with the .ordinal() method
  + Can have custom fields(properties) and methods(behaviors)

67. Encapsulation
-------------------
• Access Modifiers:
  + private ,  only inside that class
  + protected ,  inside same package or from child class
  + default ,  only inside same package (not form childs)
  + public ,  anywhere in the program
• can't narrow scope of an Overriden method
  + you can only widen or extend access level. ie.:
    - not from public -> protected
    - yes from default -> public

68. Object & JNI
-------------------
• Object:  Object is a virtual entity that has its own behavior and state which can
           distinguish it from other objects
  + That means each object will inherit methods of 'object' class
• JNI(Java Native Interface)
  + Is a foreign function interface programming framework that enables Java code running
    in Java Virtual Machine to call and be called by native applications and libraries
    written in other languages such as C or C++
  + Implemented by adding the 'native' keyword in a method

72. LSP
-------------------
""
"A child class should be able to do everything a parent clas can"
• focuses attention on types that violate polymorphism principle
◙ Problem:
   interface Bird { void fly(); void eat() }
   class Sparrow implements Bird { @Override void fly()... }
   class Ostrich implements Bird { @Override void fly()... }  //UnsupportedOperationException
  Solution:
   interface Bird { void eat(); }
   interface FlyingBird { void fly(); }
   class Sparrow implements FlyingBird { @Override void fly()... }
   class Ostrich implements Bird { @Override void eat()... }
